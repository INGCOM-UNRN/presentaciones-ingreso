<!-- _header: '¿Qué son?' -->

# Diagramas de Flujo

**Definición:**
* Representación gráfica de un algoritmo
* Usa símbolos estandarizados
* Muestra el orden con flechas

**Ventaja:**
* Se visualiza el flujo de datos
* Fácil de entender y compartir
* Detectar errores antes de programar

<!--
NOTAS DEL ORADOR:
- Herramienta de comunicación universal entre programadores.
- Permite hacer "zoom out" y ver la estructura general.
-->

---

<!-- _header: 'Símbolos básicos' -->

# Símbolo: Inicio/Fin

**Óvalo:** Marca inicio y fin del algoritmo

```
    ╭─────────╮
    │ INICIO  │
    ╰─────────╯
```

```
    ╭─────────╮
    │   FIN   │
    ╰─────────╯
```

<!--
NOTAS DEL ORADOR:
- Todo diagrama debe tener un (y solo un) Inicio.
- Puede tener múltiples Finales, pero idealmente uno solo.
-->

---

<!-- _header: 'Símbolos básicos' -->

# Símbolo: Entrada/Salida

**Paralelogramo:** Leer o mostrar datos

```
    ╱──────────╲
   ╱ Leer edad  ╲
  ╱──────────────╲
```

```
    ╱─────────────╲
   ╱ Mostrar suma  ╲
  ╱─────────────────╲
```

<!--
NOTAS DEL ORADOR:
- Interacción con el mundo exterior.
- Leer = input().
- Mostrar = print().
-->

---

<!-- _header: 'Símbolos básicos' -->

# Símbolo: Proceso

**Rectángulo:** Operaciones y cálculos

```
  ┌──────────────────┐
  │ suma = a + b     │
  └──────────────────┘
```

```
  ┌──────────────────┐
  │ contador = 1     │
  └──────────────────┘
```

<!--
NOTAS DEL ORADOR:
- Aquí ocurre la "magia" interna: cálculos, asignaciones.
- Es el cerebro del programa.
-->

---

<!-- _header: 'Símbolos básicos' -->

# Símbolo: Decisión

**Rombo:** Preguntas con Sí/No

```
        ╱╲
       ╱  ╲
      ╱edad╲
     ╱ >=18?╲
    ╱────────╲
         │
    Sí ╱   ╲ No
```

**Siempre tiene 2 salidas**

<!--
NOTAS DEL ORADOR:
- Punto de bifurcación.
- Siempre binario: Verdadero o Falso.
- No pueden salir 3 flechas de un rombo (a menos que sean casos especiales como switch, pero en diagramas básicos usamos binario).
-->

---

<!-- _header: 'Símbolos básicos' -->

# Símbolo: Flechas

**Conectores:** Muestran el orden

```
    ╭─────────╮
    │ INICIO  │
    ╰─────────╯
         │
         ↓
    ╱──────────╲
   ╱ Leer edad  ╲
  ╱──────────────╲
```

<!--
NOTAS DEL ORADOR:
- El flujo debe ser claro y seguir una dirección lógica (generalmente arriba-abajo).
-->

---

<!-- _header: 'Reglas' -->

# Reglas de Diagramas

1. Siempre empezar con INICIO
2. Siempre terminar con FIN
3. Las flechas muestran el orden
4. Los rombos tienen 2 salidas: Sí y No
5. Escribir claro dentro de cada símbolo
6. No cruzar flechas

<!--
NOTAS DEL ORADOR:
- Mantenerlo limpio. Un diagrama sucio confunde más de lo que ayuda.
-->

<!-- _header: 'Tips' -->

# Consejos para diagramas

**Hacelo simple:**
* Un proceso por rectángulo
* Una pregunta por rombo

**Mantené orden:**
* De arriba hacia abajo
* De izquierda a derecha

**Evitá:**
* Flechas que se cruzan
* Textos largos dentro de símbolos

# Ejercicio: Número positivo o negativo

**Problema:**
* Leer un número
* Decir si es positivo, negativo o cero

**¿Qué símbolos necesitás?**
* INICIO/FIN (óvalos)
* Leer número (paralelogramo)
* Decisiones (rombos) - ¿cuántos?
* Mostrar resultado (paralelogramos)

---

<!-- _header: 'Pista' -->

# Pista del ejercicio

Necesitás 2 rombos:
1. ¿número > 0? → Sí: "Positivo"
2. Si No: ¿número < 0? → Sí: "Negativo"
3. Si No: "Cero"

**Inténtalo vos primero**
