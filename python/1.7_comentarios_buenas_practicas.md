<!-- _header: '¿Qué son los comentarios?' -->

# Notas en tu código

**Comentarios:**
* Texto que Python ignora
* Para explicar qué hace el código
* Para humanos, no para la máquina

**Usa el símbolo `#`**

```python
# Esto es un comentario
print("Hola")  # Comentario al lado
```

<!-- 
NOTAS DEL ORADOR:
- El compilador/intérprete salta todo lo que esté después de `#`.
- Es como escribir notas al margen en un libro.
-->

---

<!-- _header: 'Tipos de comentarios' -->

# Comentario de una línea

**Usa `#`:**
```python
# Calcular el área de un círculo
radio = 5
area = 3.14159 * radio ** 2
print(area)  # Mostrar resultado
```

**Todo después del `#` es ignorado**

<!-- 
NOTAS DEL ORADOR:
- Se pueden usar en su propia línea o al final de una línea de código ("inline").
- Se recomienda usar inline comments con moderación.
-->

---

<!-- _header: 'Tipos de comentarios' -->

# Comentarios multilínea

**Usa comillas triples `"""`:**
```python
"""
Este es un comentario
que abarca múltiples líneas.
Útil para explicaciones largas.
"""

def calcular_promedio(numeros):
    """
    Calcula el promedio de una lista.
    
    Args:
        numeros: lista de números
    Returns:
        float: el promedio
    """
    return sum(numeros) / len(numeros)
```

<!-- 
NOTAS DEL ORADOR:
- Técnicamente son strings multilínea que no se asignan a ninguna variable, pero Python los usa como docstrings.
- Es el estándar para documentar funciones y módulos.
-->

---

<!-- _header: '¿Cuándo comentar?' -->

# Buenos vs malos comentarios

**❌ Comentarios obvios:**
```python
# Incrementar x en 1
x = x + 1

# Imprimir el nombre
print(nombre)
```

**✅ Comentarios útiles:**
```python
# Convertir temperatura a Kelvin
temp_kelvin = temp_celsius + 273.15

# IMPORTANTE: API requiere formato ISO
fecha = datetime.now().isoformat()
```

<!-- 
NOTAS DEL ORADOR:
- No traducir Python a Español línea por línea.
- "Don't state the obvious". Si el código ya dice `print(nombre)`, el comentario "Imprimir nombre" es ruido.
-->

---

<!-- _header: '¿Cuándo comentar?' -->

# Qué comentar

**Comenta el "POR QUÉ", no el "QUÉ":**

**❌ Mal:**
```python
# Multiplicar precio por 1.21
precio_final = precio * 1.21
```

**✅ Bien:**
```python
# Agregar IVA (21%)
precio_final = precio * 1.21
```

<!-- 
NOTAS DEL ORADOR:
- El "Qué" lo dice el código. El "Por qué" (la intención de negocio) solo lo sabe el programador.
- En el ejemplo, 1.21 es un "número mágico". El comentario explica qué representa. Mejor aún sería usar una constante `IVA = 0.21`.
-->

# Nombres descriptivos

**❌ Poco descriptivo:**
```python
x = 25
y = 1.75
z = x / y
```

**✅ Descriptivo:**
```python
edad = 25
altura_metros = 1.75
indice = edad / altura_metros
```

**Regla:** Nombres que explican su propósito

<!-- 
NOTAS DEL ORADOR:
- Si tenés que explicar qué hace una variable con un comentario, probablemente la variable está mal nombrada.
-->

---

<!-- _header: 'Práctica 2' -->

# snake_case para variables

**En Python usa snake_case:**
```python
# ✓ Correcto
nombre_completo = "Ana García"
precio_total = 100.50
edad_usuario = 18

# ✗ Incorrecto
NombreCompleto = "Ana"  # PascalCase (para clases)
precioTotal = 100.50    # camelCase (otros lenguajes)
```

<!-- 
NOTAS DEL ORADOR:
- PEP 8: La guía de estilo oficial de Python.
- Seguir estándares facilita leer código de otros.
-->

---

<!-- _header: 'Práctica 3' -->

# Espacios en operadores

**Para legibilidad:**
```python
# ✗ Difícil de leer
resultado=x+y*z-w/v

# ✓ Fácil de leer
resultado = x + y * z - w / v
```

**Excepción en prioridad:**
```python
# Enfatizar precedencia
resultado = x + y*z  # y*z se hace primero
```

<!-- 
NOTAS DEL ORADOR:
- "Whitespace is free". Usalo.
- Ayuda a visualizar la estructura de la operación.
-->

<!-- _header: 'Práctica 4' -->

# Líneas no muy largas

**Máximo 79-100 caracteres:**
```python
# ✗ Muy largo
mensaje = "Este es un mensaje muy largo que se extiende demasiado y es difícil de leer en la pantalla"

# ✓ Dividir en líneas
mensaje = (
    "Este es un mensaje largo "
    "dividido en múltiples líneas "
    "para mejor legibilidad"
)
```

<!-- 
NOTAS DEL ORADOR:
- Evitar el scroll horizontal. Nadie quiere scrollear para ver el final de la línea.
- Los paréntesis `()` permiten dividir líneas sin usar `\`.
-->

<!-- _header: 'Práctica 5' -->

# Una cosa por línea

**❌ Múltiples asignaciones:**
```python
x = 5; y = 10; z = 15  # Difícil de leer
```

**✅ Una por línea:**
```python
x = 5
y = 10
z = 15
```

**Más claro y fácil de depurar**

<!-- 
NOTAS DEL ORADOR:
- Aunque Python permite `;`, es muy raro verlo.

<!-- _header: 'Errores comunes' -->

# Error 1: Confundir = con ==

**Asignación (=):**
```python
edad = 18  # Guardar valor
```

**Comparación (==):**
```python
if edad == 18:  # Preguntar
    print("Tiene 18 años")
```

**Memotécnica:**
* `=` → guardar
* `==` → preguntar

<!-- 
NOTAS DEL ORADOR:
- Repaso de un clásico error.
-->

---

<!-- _header: 'Errores comunes' -->

# Error 2: No convertir input()

**❌ Error:**
```python
numero = input("Número: ")  # "5" (texto)
resultado = numero * 2
print(resultado)  # "55" (WTF?)
```

**✅ Solución:**
```python
numero = int(input("Número: "))  # 5 (número)
resultado = numero * 2
print(resultado)  # 10 ✓
```

<!-- 
NOTAS DEL ORADOR:
- "55" es la concatenación de "5" + "5".
-->

---

<!-- _header: 'Errores comunes' -->

# Error 3: División por cero

**❌ Error:**
```python
resultado = 10 / 0  # ZeroDivisionError
```

**✅ Validar:**
```python
divisor = int(input("Divisor: "))
if divisor != 0:
    resultado = 10 / divisor
    print(resultado)
else:
    print("No se puede dividir por cero")
```

<!-- 
NOTAS DEL ORADOR:
- Programación defensiva. Anticipar qué puede salir mal.
-->

---

<!-- _header: 'Errores comunes' -->

# Error 4: Indentación

**Python usa espacios para bloques:**

**❌ Error:**
```python
def saludar():
nombre = "Ana"  # Sin espacios
    print(nombre)  # Inconsistente
```

**✅ Correcto:**
```python
def saludar():
    nombre = "Ana"  # 4 espacios
    print(nombre)   # 4 espacios
```

<!-- 
NOTAS DEL ORADOR:
- En Python la indentación NO es estética, es SINTAXIS.
- 4 espacios es el estándar.
-->

# Código bien escrito

```python
# Constantes
IVA = 0.21

# Calcular precio final con IVA
precio_base = float(input("Precio base: $"))

# Aplicar impuesto
precio_con_iva = precio_base * (1 + IVA)

# Mostrar resultado
print(f"Precio base: ${precio_base:.2f}")
print(f"IVA (21%): ${precio_base * IVA:.2f}")
print(f"Total: ${precio_con_iva:.2f}")
```
