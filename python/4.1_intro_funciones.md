<!-- _header: '¿Qué es una función?' -->

# La máquina mágica

**Analogía: Licuadora**
* Le ponés frutas (argumentos)
* Apretás el botón (llamar función)
* Te devuelve un licuado (resultado)

**En programación:**
* Bloque de código con nombre
* Recibe datos de entrada
* Realiza tarea específica
* Devuelve resultado

<!--
NOTAS DEL ORADOR:
- Usar la analogía de la "Caja Negra" o "Licuadora".
- No necesitamos saber CÓMO funciona el motor de la licuadora, solo necesitamos saber cómo usarla (interfaz).
- INPUT (Frutas) -> PROCESO (Licuar) -> OUTPUT (Licuado).
- Destacar que una función encapsula una tarea.
-->

# Código repetitivo

```python
# Promedio examen 1
suma1 = 7 + 8 + 9
promedio1 = suma1 / 3
print(f"Promedio 1: {promedio1}")

# Promedio examen 2 (REPETIMOS TODO)
suma2 = 6 + 7 + 8
promedio2 = suma2 / 3
print(f"Promedio 2: {promedio2}")

# Promedio examen 3 (OTRA VEZ)
suma3 = 9 + 9 + 10
promedio3 = suma3 / 3
print(f"Promedio 3: {promedio3}")
```

<!--
NOTAS DEL ORADOR:
- Preguntar: "¿Qué pasa si ahora queremos cambiar la forma de calcular el promedio? Digamos, ponderado."
- Respuesta: "Tendríamos que cambiar el código en 3 lugares distintos."
- Principio DRY (Don't Repeat Yourself).
- Mostrar lo propenso a errores que es este enfoque (copiar/pegar a menudo lleva a olvidar cambiar nombres de variables).
-->

<!-- _header: 'Solución con funciones' -->

# Código reutilizable

```python
def calcular_promedio(notas):
    """Calcula el promedio de una lista."""
    return sum(notas) / len(notas)

# Ahora solo llamamos la función
print(f"Promedio 1: {calcular_promedio([7, 8, 9])}")
print(f"Promedio 2: {calcular_promedio([6, 7, 8])}")
print(f"Promedio 3: {calcular_promedio([9, 9, 10])}")
```

**Más corto, más claro**

<!--
NOTAS DEL ORADOR:
- Resaltar la limpieza del código.
- Explicar que `calcular_promedio` es un nombre que inventamos nosotros.
- Si hay un error en la lógica, se arregla en UN solo lugar.
- Mencionar que `sum` y `len` también son funciones (built-in).
-->

<!-- _header: 'Ventajas' -->

# ¿Por qué usar funciones?

**Reutilización:**
* Escribís el código una vez
* Lo usás muchas veces

**Organización:**
* Código más limpio
* Fácil de leer y entender

**Mantenimiento:**
* Cambios en un solo lugar
* Menos errores

<!--
NOTAS DEL ORADOR:
- Abstracción: Nos permite pensar en "calcular promedio" en lugar de "sumar y dividir".
- Divide y vencerás: Romper problemas grandes en funciones pequeñas.
- Testing: Es más fácil probar una función pequeña que un programa entero gigante.
-->

# def - definir función

**Estructura:**
```python
def nombre_funcion():
    """Documentación (opcional pero recomendado)."""
    # Código que hace algo
    instruccion1
    instruccion2
```

**Ejemplo:**
```python
def saludar():
    """Imprime un saludo."""
    print("¡Hola! Bienvenido/a a Python")
```

<!--
NOTAS DEL ORADOR:
- Explicar palabra clave `def`.
- Paréntesis `()` obligatorios aunque estén vacíos.
- Dos puntos `:` obligatorios al final.
- Indentación: CRÍTICA. Todo lo que pertenece a la función debe estar indentado.
- Docstring: Buena práctica desde el día 1.
-->

---

<!-- _header: 'Definir vs Llamar' -->

# Dos pasos diferentes

**1. Definir (escribir la receta):**
```python
def saludar():
    print("¡Hola!")
# No pasa nada aún
```

**2. Llamar (ejecutar):**
```python
saludar()  # Ahora sí se ejecuta
# Salida: ¡Hola!
```

**Definir = crear, Llamar = usar**

<!--
NOTAS DEL ORADOR:
- Analogía de la receta:
    - Definir es escribir la receta en el libro.
    - Llamar es cocinar el plato.
- Un error común de principiantes es definir la función y esperar que se ejecute sola.
- Mostrar que sin la llamada, el programa no imprime nada.
-->

<!-- _header: 'Anatomía' -->

# Partes de una función

```python
def calcular_area():
    """Calcula área de un cuadrado."""
    lado = 5
    area = lado * lado
    print(f"Área: {area}")
```

**Componentes:**
1. `def` → palabra clave
2. `calcular_area` → nombre
3. `()` → paréntesis (para parámetros)
4. `:` → dos puntos
5. Docstring (opcional)
6. Cuerpo indentado

<!--
NOTAS DEL ORADOR:
- Desglosar línea por línea.
- Variable `lado` y `area` son LOCALES (scope), mencionar brevemente que solo existen ahí dentro.
- Énfasis en los dos puntos `:`.
-->

<!-- _header: 'Nomenclatura' -->

# Nombres de funciones

**Reglas:**
* snake_case (minúsculas con _)
* Descriptivo (qué hace)
* Verbo si es acción

**✅ Buenos nombres:**
```python
def calcular_promedio()
def validar_email()
def mostrar_menu()
```

**❌ Malos nombres:**
```python
def cp()  # Muy corto
def CalcularPromedio()  # No es snake_case
def x()  # No descriptivo
```

<!--
NOTAS DEL ORADOR:
- Importancia de nombres descriptivos: el código se lee más de lo que se escribe.
- Convención PEP 8: minúsculas_con_guiones.
- Verbo + Sustantivo suele ser buena fórmula (calcular_promedio, obtener_datos).
-->

<!-- _header: 'Ejemplo simple' -->

# Función sin parámetros

```python
def mostrar_bienvenida():
    """Muestra mensaje de bienvenida."""
    print("=" * 40)
    print("   BIENVENIDO AL SISTEMA")
    print("=" * 40)

# Llamar la función
mostrar_bienvenida()
```

**Salida:**
```
========================================
   BIENVENIDO AL SISTEMA
========================================
```

<!-- _header: 'Múltiples llamadas' -->

# Reutilizar funciones

```python
def separador():
    """Imprime línea separadora."""
    print("-" * 30)

# Usar varias veces
print("Inicio")
separador()
print("Contenido")
separador()
print("Fin")
```

**Una definición, múltiples usos**

<!--
NOTAS DEL ORADOR:
- Este es el momento "Aha!": escribimos la lógica del separador una vez, la usamos tres.
- Si queremos cambiar el separador a asteriscos `*`, solo cambiamos un lugar.
-->

<!-- _header: 'Orden importa' -->

# Definir antes de llamar

**❌ Error:**
```python
saludar()  # Error: no está definida

def saludar():
    print("Hola")
```

**✅ Correcto:**
```python
def saludar():
    print("Hola")

saludar()  # Funciona
```
