# Comparar valores

**Operadores de comparación:**
* Comparan dos valores
* Responden Verdadero o Falso
* Siempre devuelven `bool`

**Es como preguntar:**
* "¿Soy más alto que Juan?"
* Respuesta: Sí (True) o No (False)

<!--
NOTAS DEL ORADOR:
- Insistir en que el resultado SIEMPRE es booleano.
- No devuelve un número, ni texto. Devuelve un estado de verdad.
-->

---

<!-- _header: 'Tipos de comparación' -->

# Los 6 operadores

**Igualdad:**
* `==` igual a
* `!=` diferente de

**Orden:**
* `>` mayor que
* `<` menor que
* `>=` mayor o igual que
* `<=` menor o igual que

<!--
NOTAS DEL ORADOR:
- Listado rápido.
- Atención a los de dos caracteres (`==`, `!=`, `>=`, `<=`). No pueden tener espacios en el medio.
-->

---

<!-- _header: 'Igualdad' -->

# Igual y diferente

**Igual a (==):**
```python
5 == 5   # True
5 == 3   # False
```

**Diferente de (!=):**
```python
5 != 3   # True
5 != 5   # False
```

**Nota:** `==` no es `=` (asignación)

<!--
NOTAS DEL ORADOR:
- Error número 1 de principiantes: Usar `=` para comparar.
- Regla mnemotécnica:
    - `=` Asigna (hace que sea igual).
    - `==` Pregunta (¿es igual?).
-->

---

<!-- _header: 'Orden' -->

# Mayor y menor

**Mayor que (>):**
```python
8 > 5    # True
3 > 10   # False
```

**Menor que (<):**
```python
3 < 10   # True
8 < 5    # False
```

<!--
NOTAS DEL ORADOR:
- Como en la escuela primaria. La boca del pacman se come al más grande.
-->

---

<!-- _header: 'Orden' -->

# Mayor/menor o igual

**Mayor o igual (>=):**
```python
5 >= 5   # True
5 >= 3   # True
3 >= 5   # False
```

**Menor o igual (<=):**
```python
3 <= 5   # True
5 <= 5   # True
8 <= 5   # False
```

<!--
NOTAS DEL ORADOR:
- Importante para condiciones de borde (ej: mayoría de edad a los 18, inclusive).
-->

---

<!-- _header: 'Pensalo así' -->

# Cada comparación es una pregunta

**Python responde True o False:**

```python
edad = 18

edad >= 18  # ¿Edad >= 18? → True
edad == 16  # ¿Edad == 16? → False
edad < 21   # ¿Edad < 21? → True
```

**Las respuestas son booleanas**

<!--
NOTAS DEL ORADOR:
- Ver el código como un diálogo.
- `edad == 16` no cambia la edad a 16, solo pregunta.
-->

---

<!-- _header: 'Ejemplo práctico' -->

# Verificando condiciones

```python
edad = 18

es_mayor = edad >= 18
print(f"¿Es mayor de edad? {es_mayor}")
# True

precio = 100
tiene_descuento = precio > 50
print(f"¿Tiene descuento? {tiene_descuento}")
# True
```

<!--
NOTAS DEL ORADOR:
- Podemos guardar el resultado de la pregunta en una variable.
- Esa variable (`es_mayor`) es de tipo `bool`.
-->

---

<!-- _header: 'Importante' -->

# Comparación vs Asignación

**Asignación (=):**
```python
edad = 18  # Guarda 18 en edad
```

**Comparación (==):**
```python
es_adulto = (edad == 18)  # True o False
```

**No confundir:** `=` con `==`

<!--
NOTAS DEL ORADOR:
- Refuerzo visual.
- La confusión `=` vs `==` es fuente de muchos bugs lógicos.
-->

---

<!-- _header: 'Con cadenas' -->

# Comparar texto

**Orden lexicográfico (diccionario):**
```python
"a" < "b"        # True
"manzana" < "pera"  # True
```

**Sensible a mayúsculas:**
```python
"Ana" == "ana"   # False
"Ana" == "Ana"   # True
```

<!--
NOTAS DEL ORADOR:
- Advertencia: En orden ASCII, las mayúsculas vienen ANTES que las minúsculas ("Z" < "a").
-->

<!-- _header: '¿Para qué?' -->

# Verificar múltiples cosas

**Ejemplo de la vida real:**
* "¿Puedo conducir?"
* Necesito: tener 18 años **Y** tener licencia
* Ambas condiciones deben cumplirse

**Operadores lógicos:**
* Combinan varias condiciones
* Una respuesta final: True o False

<!--
NOTAS DEL ORADOR:
- Lógica Booleana.
- Base de la electrónica digital y la computación.
-->

---

<!-- _header: 'Los 3 operadores' -->

# and, or, not

**and (Y):**
* Ambas condiciones deben ser True

**or (O):**
* Al menos una debe ser True

**not (NO):**
* Invierte el valor

<!--
NOTAS DEL ORADOR:
- En Python se escriben en inglés, en minúsculas.
- Son muy legibles: `if edad > 18 and tiene_entrada:`.
-->

---

<!-- _header: 'Operador and' -->

# Y lógico (and)

**Ambas deben ser True:**
```python
True and True    # True
True and False   # False
False and True   # False
False and False  # False
```

**Ejemplo:**
```python
edad = 20
tiene_licencia = True

puede_conducir = (edad >= 18) and tiene_licencia
# True (ambas son True)
```

<!--
NOTAS DEL ORADOR:
- Analogía: Para abrir una caja fuerte de dos llaves, necesito la llave A Y la llave B.
-->

---

<!-- _header: 'Operador or' -->

# O lógico (or)

**Al menos una debe ser True:**
```python
True or True     # True
True or False    # True
False or True    # True
False or False   # False
```

**Ejemplo:**
```python
es_fin_semana = True
es_feriado = False

puede_descansar = es_fin_semana or es_feriado
# True (una es True)
```

<!--
NOTAS DEL ORADOR:
- Analogía: Para entrar al cine, puedo pagar efectivo O tarjeta. Cualquiera sirve.
-->

---

<!-- _header: 'Operador not' -->

# Negación (not)

**Invierte el valor:**
```python
not True    # False
not False   # True
```

**Ejemplo:**
```python
esta_lloviendo = False
puede_salir = not esta_lloviendo
# True (invierte False)
```

<!--
NOTAS DEL ORADOR:
- El operador rebelde.
- Útil para decir "mientras NO termine" o "si NO está vacío".
-->

---

edad = 25

puede_viajar = (edad >= 18) and \

<!--
NOTAS DEL ORADOR:

<!-- _header: 'Precedencia' -->

# Orden de evaluación

**Prioridad:**
1. `not`
2. `and`
3. `or`

**Ejemplo:**
```python
True or False and False
# Evalúa: True or (False and False)
# Resultado: True
```

**Usa paréntesis para evitar confusión**

<!--
NOTAS DEL ORADOR:
- Como en matemática (multiplicación antes que suma), aquí `and` antes que `or`.
- Pero no confíes en tu memoria, usá paréntesis.
-->

---

<!-- _header: 'Paso a paso' -->

# Evaluación paso a paso

**Expresión: `(5 > 3) and not (4 == 4)`**

1. **Paréntesis:** `(True) and not (True)`
   `True and not True`

2. **NOT:** `not True` es `False`
   `True and False`

3. **AND:** `True and False` es `False`
   **Resultado: False**

NOTAS DEL ORADOR:
- Este ejemplo visual es crucial para desmitificar cómo la computadora "piensa".
- Reducción sucesiva hasta llegar al valor final.
