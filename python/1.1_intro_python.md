<!-- _header: '¿Por qué Python?' -->

# Python es como el español

**Comparación con otros lenguajes:**

**Java (complicado):**
```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("¡Hola Mundo!");
    }
}
```

**Python (¡simple!):**
```python
print("¡Hola Mundo!")
```

<!--
NOTAS DEL ORADOR:
- Este contraste visual es poderoso. No necesitás explicar qué hace el código de Java, solo mostrar la diferencia de complejidad.
- Python elimina el "ceremonial" de otros lenguajes (clases obligatorias, métodos main, etc.) y te deja concentrar en la lógica.
- Dato curioso: El creador de Python, Guido van Rossum, diseñó el lenguaje para que fuera "tan fácil de leer como pseudocódigo".
-->

# Disección línea por línea

```python
print("¡Hola Mundo!")
```

**Componentes:**
* `print` → función (comando que hace algo)
* `(...)` → paréntesis indican función
* `"¡Hola Mundo!"` → texto a mostrar

**Función = orden a la computadora**

<!--
NOTAS DEL ORADOR:
- La terminología importa. "Función" puede sonar abstracto, pero si lo pensás como "orden" o "comando" se vuelve más claro.
- Más adelante aprenderán a crear sus propias funciones, pero por ahora usan las que Python provee (como print).
- Notar el código de colores: Funciones en un color, Texto en otro. Ayuda visual.

<!-- _header: 'Detalles' -->

# Sobre las comillas

**Python acepta ambas:**
```python
print("Hola")    # Comillas dobles
print('Hola')    # Comillas simples
```

**Ambas funcionan igual**

**Consejo:**
* Elegí una y sé consistente
* Usá dobles si el texto tiene apóstrofo
* Ejemplo: `"It's working!"`

<!--
NOTAS DEL ORADOR:
- La consistencia es parte de las buenas prácticas de programación. PEP 8 (la guía de estilo de Python) recomienda comillas dobles, pero lo más importante es ser consistente.
- El caso del apóstrofo es práctico: "It's working!" funciona con dobles, pero 'It's working!' causaría un error de sintaxis porque Python pensaría que la cadena termina en el apóstrofo.
-->

<!-- _header: 'Variaciones' -->

# Experimentá con print()

**Múltiples argumentos:**
```python
print("Hola", "Mundo")
# Salida: Hola Mundo
```

**Con números:**
```python
print("Tengo", 18, "años")
# Salida: Tengo 18 años
```

**Separador personalizado:**
```python
print("A", "B", "C", sep="-")
# Salida: A-B-C
```

<!--
NOTAS DEL ORADOR:
- Estas variaciones muestran que print() es más flexible de lo que parece. No necesitan memorizar todas las opciones ahora, pero es bueno que sepan que existen.
- El parámetro 'sep' (separator) es útil cuando quieren formatear salidas. Por defecto, print() separa con un espacio.
-->

<!-- _header: 'Errores comunes' -->

# Errores típicos de principiante

**Error 1: Olvidar comillas**
```python
print(Hola)  # ❌ Error
print("Hola")  # ✅ Correcto
```

**Error 2: Olvidar paréntesis**
```python
print "Hola"  # ❌ Error (Python 2)
print("Hola")  # ✅ Correcto
```

**Error 3: Comillas disparejas**
```python
print("Hola')  # ❌ Error
print("Hola")  # ✅ Correcto
```

<!--
NOTAS DEL ORADOR:
- Ver errores desde el principio es pedagógico. No quieras "protegerlos" de los errores; al contrario, enseñales a leerlos y entenderlos.
- El primer error de sintaxis que vean puede ser intimidante, pero explicá que los mensajes de error son amigos que te dicen exactamente qué está mal y dónde.
- Consejo: Cuando alguien tenga un error, pedile que LEA el mensaje completo en voz alta. Muchas veces la solución está en el mensaje mismo.
-->
