# Como botones de calculadora

**Operadores:**
* Símbolos que indican operaciones
* Le dicen a Python qué hacer
* Como botones: +, -, ×, ÷

**Python tiene:**
* Los que conocés de matemática
* Y algunos extras muy útiles

<!--
NOTAS DEL ORADOR:
- Concepto de "operando" y "operador". `a + b`: `a` y `b` son operandos, `+` es el operador.
-->

<!-- _header: 'Operadores básicos' -->

# Las 4 operaciones

**Suma (+):**
```python
5 + 3  # 8
```

**Resta (-):**
```python
10 - 4  # 6
```

**Multiplicación (*):**
```python
6 * 7  # 42
```

**División (/):**
```python
15 / 3  # 5.0
```

<!--
NOTAS DEL ORADOR:
- El asterisco `*` se usa para multiplicar porque la `x` se confunde con la variable x.
- La barra `/` es la división estándar.
-->

---

<!-- _header: 'División' -->

# Nota sobre división

**La división SIEMPRE da float:**
```python
15 / 3   # 5.0 (no 5)
10 / 2   # 5.0 (no 5)
7 / 2    # 3.5
```

**Incluso si el resultado es entero:**
* Retorna `5.0` en vez de `5`
* Es el comportamiento estándar

<!--
NOTAS DEL ORADOR:
- Esto es específico de Python 3. En Python 2 era diferente (y causaba muchos bugs).
- Python "promueve" el resultado a flotante para no perder información accidentalmente.
-->

---

<!-- _header: 'Operadores extras' -->

# División entera (//)

**Divide y descarta decimales:**
```python
15 // 3  # 5 (entero)
7 // 2   # 3 (no 3.5)
10 // 3  # 3 (no 3.333...)
```

**Útil cuando:**
* Solo querés la parte entera
* Contar grupos completos
* Ejemplo: ¿Cuántos equipos de 5?

<!--
NOTAS DEL ORADOR:
- Doble barra `//`.
- Se llama "floor division" (división piso) porque redondea hacia abajo.
- Ejemplo: Si tengo 7 caramelos para 2 personas, le tocan 3 a cada una. (Y sobra 1, que veremos ahora).
-->

---

<!-- _header: 'Operadores extras' -->

# Módulo (%)

**Da el resto de la división:**
```python
7 % 2   # 1 (resto de 7÷2)
10 % 3  # 1 (resto de 10÷3)
15 % 5  # 0 (divisible exacta)
```

**Usos comunes:**
* Saber si es par o impar
* Ciclos repetitivos
* Validaciones

<!--
NOTAS DEL ORADOR:
- El operador más subestimado. Es súper útil.
- No es porcentaje. Es "Módulo" o "Resto".
- Analogía del reloj: 13 horas son las 1 (13 % 12 = 1).
-->

---

<!-- _header: 'Módulo - Ejemplo' -->

# Par o impar

**Técnica:**
* Si `numero % 2 == 0` → par
* Si `numero % 2 == 1` → impar

**Ejemplos:**
```python
6 % 2  # 0 → par
7 % 2  # 1 → impar
8 % 2  # 0 → par
9 % 2  # 1 → impar
```

<!--
NOTAS DEL ORADOR:
- Este es un patrón clásico de programación.
- `if n % 2 == 0:` es la forma estándar de preguntar "¿Es par?".
-->

---

<!-- _header: 'Operadores extras' -->

# Potencia (**)

**Elevar a una potencia:**
```python
2 ** 3   # 8 (2×2×2)
5 ** 2   # 25 (5×5)
10 ** 3  # 1000
3 ** 0   # 1
```

**No es:** `2^3` (eso es otra cosa)
**Es:** `2**3`

<!--
NOTAS DEL ORADOR:
- Ojo: En muchas calculadoras es `^`. En Python `^` es XOR (operador bit a bit), que hace algo totalmente distinto.
- Error común: Escribir `2^3` y que dé `1` en lugar de `8`.
-->

<!-- _header: 'Precedencia' -->

# Orden de operaciones

**Como en matemática:**
1. Paréntesis `()`
2. Potencia `**`
3. Multiplicación, División `*`, `/`, `//`, `%`
4. Suma, Resta `+`, `-`

**Ejemplos:**
```python
2 + 3 * 4      # 14 (no 20)
(2 + 3) * 4    # 20
2 ** 3 + 1     # 9 (no 16)
```

<!--
NOTAS DEL ORADOR:
- PAPOMUDAS (Paréntesis, Potencia, Multiplicación, División, Adición, Sustracción).
- Si tenés dudas, usá paréntesis.
-->

---

<!-- _header: 'Paso a paso' -->

# Precedencia en acción

**Resolvamos: `10 + 2 * 3 ** 2`**

1. **Potencia primero:** `3 ** 2` es 9
   `10 + 2 * 9`

2. **Multiplicación después:** `2 * 9` es 18
   `10 + 18`

3. **Suma al final:** `10 + 18` es 28
   **Resultado: 28**

# Usa paréntesis

**Para claridad:**
```python
# Confuso
resultado = 2 + 3 * 4 / 2

# Más claro
resultado = 2 + ((3 * 4) / 2)
```

**Aunque no sean necesarios:**
* Hacen el código más legible
* Evitan errores de interpretación

# Variables y operadores

```python
a = 10
b = 3

suma = a + b       # 13
resta = a - b      # 7
producto = a * b   # 30
division = a / b   # 3.333...
division_entera = a // b  # 3
resto = a % b      # 1
potencia = a ** b  # 1000
```

<!--
NOTAS DEL ORADOR:
- Las variables se comportan como los números que contienen.
-->

<!-- _header: 'Asignación compuesta' -->

# Atajos útiles

**Asignación con operación:**
```python
x = 5
x = x + 3  # 8

# Atajo:
x = 5
x += 3     # 8 (igual que x = x + 3)
```

**Otros atajos:**
```python
x -= 2   # x = x - 2
x *= 3   # x = x * 3
x /= 2   # x = x / 2
```

<!--
NOTAS DEL ORADOR:
- "Syntactic Sugar" (Azúcar sintáctico).
- No agrega funcionalidad nueva, solo hace más dulce (fácil) escribirlo.
- Muy común en contadores (`i += 1`).
-->

# Ejemplo de solución

print(f"{a} // {b} = {a // b}")
print(f"{a} % {b} = {a % b}")
print(f"{a} ** {b} = {a ** b}")

<!-- _header: 'Errores comunes' -->

# Errores típicos

**Error 1: División por cero**
```python
10 / 0  # ZeroDivisionError
```

**Error 2: Confundir operadores**
```python
2 ^ 3  # NO es potencia (es XOR)
2 ** 3  # Sí es potencia
```

**Error 3: Tipos incompatibles**
```python
"Hola" * "Mundo"  # TypeError
```

<!--
NOTAS DEL ORADOR:
- Advertir sobre ZeroDivisionError: Es fatal, detiene el programa.
- Explicar que multiplicar strings sí funciona (`"a" * 3`), pero string * string no.
-->
